**录制**就是把 ROS 运行时的消息（topic）保存成文件，之后可以回放复现。

在 ROS 里通常叫 **rosbag 录制**。

---

## 这个项目里 "录制" 指的是什么？

`record_gazebo.launch` 里的内容：

```xml
<node pkg="rosbag" type="record" name="rosbag_record_gazebo"
      args="/stereo_camera/left/camera_info
            /stereo_camera/left/image_rect_color
            /stereo_camera/right/camera_info
            /stereo_camera/right/image_rect_color
            /velodyne_points
            /tf
            /tf_static" />
```

它会启动一个 `rosbag record` 节点，把 Gazebo 仿真发布的话题存到一个 `.bag` 文件里。

---

## 有什么用？

| 用途       | 说明                       |
| -------- | ------------------------ |
| **离线调试** | 录一次，之后反复回放，不用一直开 Gazebo  |
| **复现实验** | 同样的数据可以重复跑标定算法，方便对比      |
| **分享数据** | 把 `.bag` 发给别人，别人也能复现你的场景 |
| **保存真值** | 仿真里可以记录传感器的真实姿态，用于评估     |

---

## 录下来的文件在哪？

运行后会在当前目录生成类似：

```
2026-06-30-10-00-00.bag
```

你可以在启动时指定文件名：

```bash
rosbag record -O my_scene /velodyne_points /stereo_camera/left/image_rect_color
```

---

## 怎么回放？

```bash
rosbag play my_scene.bag
```

RViz 里就能看到和当时一样的点云和图像。

---

## 在这个项目里的位置

`record_gazebo.launch` 只是一个通用录制脚本。

`journal/record/` 下的 launch 更具体，例如：

```bash
record_mono_hdl64.launch
record_stereo.launch
record_vlp16.launch
```

它们对应不同传感器配置。
