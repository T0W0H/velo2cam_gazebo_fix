## 1. 录制怎么用？

### 方式一：用项目里的 launch 文件

```bash
roslaunch velo2cam_gazebo record_gazebo.launch
```

这会在后台启动 `rosbag record`，录制的 topic 在 `record_gazebo.launch` 里已经写死了：

```xml
/stereo_camera/left/camera_info
/stereo_camera/left/image_rect_color
/stereo_camera/right/camera_info
/stereo_camera/right/image_rect_color
/velodyne_points
/tf
/tf_static
```

录好的 `.bag` 文件会存在你执行命令的目录。

### 方式二：手动指定 topic

如果你只想录某几个 topic：

```bash
rosbag record /velodyne_points /stereo_camera/left/image_rect_color
```

指定文件名：

```bash
rosbag record -O my_data /velodyne_points /stereo_camera/left/image_rect_color
```

`-O` 后面跟输出文件名。

### 常用选项

| 选项 | 作用 |
|------|------|
| `-O name` | 指定输出文件名 |
| `-a` | 录制所有 topic |
| `-j` | 压缩 |
| `-b 1024` | 设置缓冲区大小 |
| `--split --duration=60s` | 每 60 秒拆分一个 bag |

---

## 2. 什么情景下用？

| 场景 | 为什么录 |
|------|---------|
| **仿真一次，反复调试** | Gazebo 启动慢、吃资源，录一次可以反复回放 |
| **算法对比** | 同一组数据跑不同参数/算法，公平比较 |
| **没有 Gazebo 也能工作** | 在另一台电脑上只跑 rosbag，不跑仿真 |
| **提交实验结果** | 把 `.bag` 作为实验数据保存下来 |
| **复现 bug** | 把异常时刻的数据录下来，方便定位 |

---

## 3. 可以与其他的一起开吗？

**可以。** `record_gazebo.launch` 只录制，不生成模型、不启动 Gazebo，所以能和其他功能 launch 同时开。

### 正确组合方式

```bash
# 终端 1：启动仿真
roslaunch velo2cam_gazebo mono_hdl64_p1_real.launch

# 终端 2：开始录制
roslaunch velo2cam_gazebo record_gazebo.launch
```

或者一条命令里同时启动（但通常分两个终端更方便）：

```bash
roslaunch velo2cam_gazebo mono_hdl64_p1_real.launch & \
roslaunch velo2cam_gazebo record_gazebo.launch
```

### 不能这样组合

```bash
roslaunch velo2cam_gazebo mono_hdl64_p1_real.launch
roslaunch velo2cam_gazebo mono_mono_p1_real.launch
```

两个场景 launch 会冲突。

---

## 4. 完整示例流程

```bash
# 1. 启动 Gazebo 仿真
roslaunch velo2cam_gazebo mono_hdl64_p1_real.launch

# 2. 等 Gazebo 稳定后，开始录数据
roslaunch velo2cam_gazebo record_gazebo.launch

# 3. 录够了按 Ctrl+C 停止 rosbag

# 4. 查看录到的内容
rosbag info 2026-06-30-xx-xx-xx.bag

# 5. 离线回放
rosbag play 2026-06-30-xx-xx-xx.bag
```

---

## 5. 注意事项

- 录之前确保 topic 已经在发布，否则只录到空数据
- `.bag` 文件可能很大，注意磁盘空间
- 录制时 topic 名要写对，否则录不到

---

## 总结

> **录制就是“把 ROS 话题存成文件”，可以和 Gazebo 仿真一起开；但多个场景 launch 不能一起开。**


### ==tip:==

**边标定边录制是很常见的做法**。

标定节点和 `rosbag record` 都是订阅同一个 topic，**互不干扰**。

---

## 为什么可以？

- `rosbag record` 只是**监听并保存话题数据**
- 标定算法也是**订阅话题进行处理**
- 两者是并行的，录制的不会吃掉数据，标定的也能正常收到

---

## 操作方式

### 开三个终端

```bash
# 终端 1：启动仿真
roslaunch velo2cam_gazebo mono_hdl64_p1_real.launch

# 终端 2：启动标定
roslaunch velo2cam_calibration velo2cam_calibration.launch

# 终端 3：开始录制
roslaunch velo2cam_gazebo record_gazebo.launch
```

或者先启动标定，再启动录制，顺序无所谓。

---

## 这样做的好处

| 好处         | 说明                      |
| ---------- | ----------------------- |
| **保留原始数据** | 万一标定结果有问题，可以回放 bag 重新分析 |
| **复现实验**   | 同样的输入数据，跑不同版本算法对比       |
| **调试方便**   | 标定过程可能出错，有 bag 可以离线复现   |
| **保存中间结果** | 可以看到标定过程中 topic 的变化     |

---

## 注意事项

1. **先等 Gazebo 稳定再开始录**
   - Gazebo 加载模型需要时间，开始录制太早可能录到空数据

2. **bag 文件会很大**
   - 点云和图像数据量大，注意磁盘空间

3. **录制时间**
   - 标定完成后按 `Ctrl+C` 停止录制即可

4. **topic 名要对**
   - 用 `rostopic list` 确认 `/velodyne_points`、图像等话题存在

---

## 总结

> **完全可以。一个跑仿真、一个跑标定、一个录数据，三个并行不冲突。**
